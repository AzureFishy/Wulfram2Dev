    UPDATE_STATS (0x1C)¶

    Overview¶

    The UPDATE_STATS packet updates player statistics in the game roster. This packet is sent from the server to clients when a player's stats change (kills, deaths, score, etc.).

    Packet Structure¶

    UPDATE_STATS Packet Structure:
┌──────────┬──────────────┬──────────────┬──────────────┬──────────────┐
│  TYPE    │   LENGTH     │  FIELD_1     │  FIELD_2     │  STATS...    │
│  0x1C    │  (2 bytes)   │  (4 bytes)   │  (4 bytes)   │  (variable)  │
└──────────┴──────────────┴──────────────┴──────────────┴──────────────┘

    Field Layout¶

    Offset Size Type Description
    0 1 uint8 Packet type: 0x1C
    1 2 uint16 BE Payload length
    3 4 int32 BE Field 1 (possibly player_id)
    7 4 int32 BE Field 2 (possibly object_id)
    11 2 uint16 BE Stat field 1
    13 2 uint16 BE Stat field 2
    15 2 uint16 BE Stat field 3
    17 2 uint16 BE Stat field 4
    19 2 uint16 BE Stat field 5
    21 8 fixed64 Extended stat 1 (position/score?)
    29 8 fixed64 Extended stat 2
    37 4 int32 BE Final field

    Total payload size: ~38 bytes (variable)

    Python Implementation¶

    def create_update_stats(player_id, object_id, stats):
    """
    Create an UPDATE_STATS packet

    Args:
        player_id: Player identifier
        object_id: Object/entity identifier
        stats: Dictionary with stat fields
    """
    payload = b''
    payload += struct.pack('>i', player_id)
    payload += struct.pack('>i', object_id)
    payload += struct.pack('>H', stats.get('stat1', 0))
    payload += struct.pack('>H', stats.get('stat2', 0))
    payload += struct.pack('>H', stats.get('stat3', 0))
    payload += struct.pack('>H', stats.get('stat4', 0))
    payload += struct.pack('>H', stats.get('stat5', 0))
    payload += struct.pack('>q', int(stats.get('extended1', 0) * 65536))
    payload += struct.pack('>q', int(stats.get('extended2', 0) * 65536))
    payload += struct.pack('>i', stats.get('final', 0))

    return make_packet(0x1C, payload)

    Notes¶

    • This packet is critical for maintaining roster state
    • The uint16 fields likely represent kills, deaths, assists, score components
    • Extended fields may contain floating-point values encoded as 64-bit fixed point
    • Further packet capture analysis needed to confirm field meanings
