    Wulfram 2 Server Implementation Guide¶

    Part 4-4: BEHAVIOR Packet - Vehicle Type Data¶

    Overview¶

    The Vehicle Type Data section transmits physics and movement parameters for the 2 playable vehicle types: Tank and Scout (Medic).

    Historical Note: Wulfram 2 was designed to support additional vehicles (Bomber, Transport, Assault Platform), but these were either gameplay-broken (Bomber was too overpowered) or incomplete. In the released game, only Tank and Scout are playable. The client still registers exactly 2 vehicle types.

    The Handler Code¶

    undefined4 *puVar1;
    int iVar2;
    undefined4 *puVar3;

    FUN_004e5080();                                // Initialize context
    FUN_004e5090(param_1);                         // Set context pointer

        FUN_004e51d0(iVar2 + 0x20);    // 4 bytes: fixed-point (field 6)

    Vehicle Type Count: 2 (Fixed)¶

    The client registers exactly 2 vehicle types during initialization:

    From FUN_004e5b50 (lines 157814-157815): 

    FUN_004e5ad0();    // Register vehicle type 0 (Tank)
FUN_004e5ad0();    // Register vehicle type 1 (Scout)

    The registration function FUN_004e5ad0 adds each vehicle to a linked list, which FUN_004e5ce0 then iterates through.

    Important: The linked list is traversed in registration order. Since FUN_004e5ad0() is called for Tank first (index 0), then Scout (index 1), the server must send Tank data before Scout data. Reversing this order will cause the client to apply Tank physics to Scout and vice versa.

    Field Purpose Analysis (Speculative)¶

    Based on code analysis patterns, these fields likely control:

    Field Default Tank Scout Likely Purpose
    1 20.0 20.0 20.0 Max speed or base velocity multiplier
    2 4.0 4.0 4.0 Acceleration rate
    3 700 700 700 Turn rate or rotation speed
    4 550 550 550 Unknown (consistent between vehicles)
    5 1.0 0.8 0.8 Handling/traction factor
    6 0.5 0.05 0.15 Braking/friction coefficient
    7 2.0 1.3 1.1 Mass/inertia factor
    8 0 0 0 Timer/cooldown related
    9 0 33000 13000 Fuel/energy capacity

    Note: These purposes are inferred from code context and may not be exact. The Scout's higher Field 6 (0.15 vs 0.05) and lower Field 9 (13000 vs 33000) align with it being a faster, lighter vehicle with less fuel capacity.

    Building Vehicle Type Data in Python¶

    import struct

def encode_fixed_point(float_value):
    """Convert a float/double to fixed-point format (4 bytes big-endian)."""
    wire_value = int(float_value * 65536)
    return struct.pack('>i', wire_value)

def create_vehicle_type_data(field1=20.0, field2=4.0, field3=700, field4=550,
                              field5=1.0, field6=0.5, field7=2.0, 
                              field8=0, field9=0):
    """
    Create data for a single vehicle type (36 bytes).

    Args:
        field1-field7: Physics parameters (doubles, sent as fixed-point)
        field3-field4, field8-field9: Integer parameters (uint32)

    Returns:
        36 bytes of vehicle type data
    """
    data = bytearray()

    # Field 1: fixed-point (4 bytes)
    data.extend(encode_fixed_point(field1))

    # Field 2: fixed-point (4 bytes)
    data.extend(encode_fixed_point(field2))

    # Field 3: uint32 big-endian (4 bytes)
    data.extend(struct.pack('>I', field3))

    # Field 4: uint32 big-endian (4 bytes)
    data.extend(struct.pack('>I', field4))

    # Field 5: fixed-point (4 bytes)
    data.extend(encode_fixed_point(field5))

    # Field 6: fixed-point (4 bytes)
    data.extend(encode_fixed_point(field6))

    # Field 7: fixed-point (4 bytes)
    data.extend(encode_fixed_point(field7))

    # Field 8: uint32 big-endian (4 bytes)
    data.extend(struct.pack('>I', field8))

    # Field 9: uint32 big-endian (4 bytes)
    data.extend(struct.pack('>I', field9))

    assert len(data) == 36, f"Vehicle type should be 36 bytes, got {len(data)}"
    return bytes(data)

def create_behavior_vehicle_type_data():
    """
    Create the complete Vehicle Type Data section (2 vehicles = 72 bytes).

    Returns:
        72 bytes of vehicle type data
    """
    data = bytearray()

    # Vehicle Type 0: Tank
    # Uses defaults except: field5=0.8, field6=0.05, field7=1.3, field9=33000
    tank_data = create_vehicle_type_data(
        field1=20.0,
        field2=4.0,
        field3=700,
        field4=550,
        field5=0.8,
        field6=0.05,
        field7=1.3,
        field8=0,
        field9=33000
    )
    data.extend(tank_data)

    # Vehicle Type 1: Scout (Medic)
    # Uses defaults except: field5=0.8, field6=0.15, field7=1.1, field9=13000
    scout_data = create_vehicle_type_data(
        field1=20.0,
        field2=4.0,
        field3=700,
        field4=550,
        field5=0.8,
        field6=0.15,
        field7=1.1,
        field8=0,
        field9=13000
    )
    data.extend(scout_data)

    assert len(data) == 72, f"Vehicle type data should be 72 bytes, got {len(data)}"
    return bytes(data)

    Simplified Version (Default Values)¶

    For initial testing, you can use default values for both vehicles:

    def create_default_vehicle_type_data():
    """Create 72 bytes using default values for both vehicle types."""
    data = bytearray()

    # Both vehicles with defaults
    for _ in range(2):
        data.extend(create_vehicle_type_data())  # All defaults

    return bytes(data)

    The 2 Playable Vehicle Types¶

    Index Vehicle Name Role Notes
    0 Tank Main combat vehicle Default spawn, heavy armor
    1 Scout Fast reconnaissance / Medic Can heal teammates

    Note: Other vehicles (Bomber, Transport, Assault Platform) exist in entity definitions but were never enabled for player use. The Bomber was tested but deemed too overpowered.

    Common Mistakes¶

    Mistake Symptom Fix
    Wrong vehicle count Data misalignment Send exactly 2 vehicle types
    Wrong bytes per vehicle Corruption of next section Send exactly 36 bytes each
    Little-endian integers Garbled values Use big-endian (>I)
    Missing fixed-point conversion Physics values 65536× wrong Multiply floats by 65536
    Sending more vehicles Client reads garbage Only Tank and Scout are registered

    Verification Checklist¶

    Before moving to Section 5, verify:

    • [ ] Exactly 2 vehicle types (Tank, then Scout)
    • [ ] Exactly 36 bytes per vehicle type
    • [ ] Total size is exactly 72 bytes
    • [ ] Fields 1, 2, 5, 6, 7: fixed-point encoded (4 bytes each)
    • [ ] Fields 3, 4, 8, 9: uint32 big-endian (4 bytes each)
    • [ ] Order: Tank first (index 0), Scout second (index 1)
