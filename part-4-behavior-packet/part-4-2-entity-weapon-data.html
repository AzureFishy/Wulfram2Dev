    Wulfram 2 Server Implementation Guide¶

    Part 4-2: BEHAVIOR Packet - Entity Weapon Data¶

    Overview¶

    The Entity Weapon Data section is the first variable-length section of the BEHAVIOR packet. It configures weapon loadouts for each playable vehicle type in the game.

    Position in BEHAVIOR Packet:
┌─────────────────────────────────────────────────────────────────────────┐
│  SECTION 1: Fixed Header (95 bytes)                        ✓ Complete  │
├─────────────────────────────────────────────────────────────────────────┤
│  SECTION 2: Entity Weapon Data (2,340 bytes)               ◄ YOU ARE   │
│             4 entities × 13 weapon slots × 45 bytes            HERE    │
├─────────────────────────────────────────────────────────────────────────┤
│  SECTION 3: Entity Core Stats (468 bytes)                              │
│  SECTION 4: Vehicle Type Data (variable)                               │
│  SECTION 5: Physics Tables + Vehicle Models (variable)                 │
│  PACKET FINALIZATION: Byte Alignment                                   │
└─────────────────────────────────────────────────────────────────────────┘

    Important: The client expects weapon data for exactly 4 entity types. These correspond to the playable vehicle types:

    Entity Index Entity Name Description
    0 Tank Main battle vehicle
    1 Scout Fast reconnaissance vehicle
    2 Medic / Assault Platform Support vehicle
    3 Bomber Air attack vehicle

    Section Size Calculation¶

    Entity Weapon Data Size:
┌─────────────────────────────────────────────────────────────────────────┐
│  4 entities × 13 slots × 45 bytes = 2,340 bytes                        │
└─────────────────────────────────────────────────────────────────────────┘

Breakdown per entity (585 bytes):
  - 13 weapon slots
  - Each slot: 45 bytes

Breakdown per weapon slot (45 bytes):
  - 5 boolean flags:    5 × 1 byte  =  5 bytes
  - 1 fixed-point:      1 × 4 bytes =  4 bytes
  - 5 integers:         5 × 4 bytes = 20 bytes
  - 4 fixed-points:     4 × 4 bytes = 16 bytes
  - Total:                          = 45 bytes

    Building Entity Weapon Data in Python¶

    import struct

def encode_fixed_point(float_value):
    """Convert a float to fixed-point format (4 bytes big-endian)"""
    wire_value = int(float_value * 65536)
    return struct.pack('>i', wire_value)

def create_weapon_slot(enabled=False, flags=(False, False, False, False),
                       param1=0.0, ints=(0, 0, 0, 0, 0),
                       doubles=(0.0, 0.0, 0.0, 0.0)):
    """
    Create a single weapon slot (45 bytes).

    Args:
        enabled: Whether this weapon slot is active
        flags: Tuple of 4 additional boolean flags
        param1: First fixed-point parameter
        ints: Tuple of 5 integer parameters
        doubles: Tuple of 4 fixed-point double parameters

    Returns:
        45 bytes of weapon slot data
    """
    data = bytearray()

    # 5 boolean flags (5 bytes)
    data.append(0x01 if enabled else 0x00)
    for flag in flags:
        data.append(0x01 if flag else 0x00)

    # 1 fixed-point double (4 bytes)
    data.extend(encode_fixed_point(param1))

    # 5 uint32 integers (20 bytes)
    for value in ints:
        data.extend(struct.pack('>I', value))

    # 4 fixed-point doubles (16 bytes)
    for value in doubles:
        data.extend(encode_fixed_point(value))

    assert len(data) == 45, f"Weapon slot should be 45 bytes, got {len(data)}"
    return bytes(data)

def create_entity_weapons(slot_configs=None):
    """
    Create weapon data for a single entity (13 slots = 585 bytes).

    Args:
        slot_configs: List of 13 slot configurations, or None for defaults

    Returns:
        585 bytes of entity weapon data
    """
    data = bytearray()

    for slot_idx in range(13):
        if slot_configs and slot_idx < len(slot_configs):
            slot_data = create_weapon_slot(**slot_configs[slot_idx])
        else:
            # Default: disabled weapon slot
            slot_data = create_weapon_slot()
        data.extend(slot_data)

    assert len(data) == 585, f"Entity weapons should be 585 bytes, got {len(data)}"
    return bytes(data)

def create_behavior_entity_weapon_data():
    """
    Create the complete Entity Weapon Data section (4 entities = 2,340 bytes).

    Returns:
        2,340 bytes of entity weapon configuration data
    """
    data = bytearray()

    # Entity 0: Tank
    data.extend(create_entity_weapons())

    # Entity 1: Scout
    data.extend(create_entity_weapons())

    # Entity 2: Medic/Assault Platform
    data.extend(create_entity_weapons())

    # Entity 3: Bomber
    data.extend(create_entity_weapons())

    assert len(data) == 2340, f"Entity weapon data should be 2340 bytes, got {len(data)}"
    return bytes(data)

    Default Values (All Zeros)¶

    For a basic implementation, you can send all weapon slots as disabled (zeros):

    def create_empty_weapon_data():
    """Create 2,340 bytes of empty weapon data (all disabled)."""
    return bytes(2340)  # All zeros

    This creates valid data where: - All 5 booleans = 0x00 (false) - All fixed-point values = 0x00000000 (0.0) - All integers = 0x00000000 (0)

    Post-Processing: FUN_004e27d0¶

    After reading all 13 weapon slots for an entity, the client calls FUN_004e27d0 to count up enabled weapons:

    // From FUN_004e27d0 (lines 155372-155403)
void FUN_004e27d0(int param_1) {
    *in_EAX = 0;      // Counter 1
    in_EAX[1] = 0;    // Counter 2
    in_EAX[2] = 0;    // Counter 3
    in_EAX[3] = 0;    // Counter 4

    pcVar1 = (char *)(param_1 + 0x11);
    iVar2 = 0xd;  // 13 iterations

    do {
        if (pcVar1[-1] != '\0') {   // If slot enabled (offset 0x10)
            if (*pcVar1 != '\0') {   // Check flag at offset 0x11
                *in_EAX = *in_EAX + 1;
            }
            if (pcVar1[1] != '\0') { // Check flag at offset 0x12
                in_EAX[1] = in_EAX[1] + 1;
            }
            if (pcVar1[2] != '\0') { // Check flag at offset 0x13
                in_EAX[2] = in_EAX[2] + 1;
            }
            if (pcVar1[3] != '\0') { // Check flag at offset 0x14
                in_EAX[3] = in_EAX[3] + 1;
            }
        }
        pcVar1 = pcVar1 + 0x58;  // Next slot
        iVar2 = iVar2 + -1;
    } while (iVar2 != 0);
}

    This counts how many weapon slots have each flag set, which is used for UI display and game logic.

    Common Mistakes¶

    Mistake Symptom Fix
    Wrong entity count Data misalignment, crash Send exactly 4 entities
    Wrong slot count Missing/extra data Send exactly 13 slots per entity
    Wrong byte count per slot Corruption of following sections Ensure 45 bytes per slot
    Little-endian integers Garbled values Use big-endian (>I)
    Forgetting fixed-point Values 65536× too small Multiply floats by 65536

    Verification Checklist¶

    Before moving to Section 3, verify:

    • [ ] Exactly 4 entities worth of data
    • [ ] Exactly 13 weapon slots per entity
    • [ ] Exactly 45 bytes per weapon slot
    • [ ] Total size is exactly 2,340 bytes
    • [ ] All integers are big-endian
    • [ ] All floats/doubles are fixed-point encoded
    • [ ] Booleans are 1 byte each (0x00 or 0x01)
