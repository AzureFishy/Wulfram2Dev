    UDP WEAPONS (0x0C) and ACK_WPNS (0x0D)¶

    Overview¶

    WEAPONS packets synchronize weapon state between server and clients. This includes ammunition counts, reload states, and weapon slot configurations.

    WEAPONS (0x0C) Structure¶

    WEAPONS Packet Structure:
┌──────────┬──────────────┬────────────────────────────────┬─────────────────────┐
│  TYPE    │  OBJECT_ID   │  WEAPON_STRINGS (13×)          │  AMMO_ARRAYS        │
│  0x0C    │  (4 bytes)   │  (13 length-prefixed strings)  │  (52 bytes each)    │
└──────────┴──────────────┴────────────────────────────────┴─────────────────────┘
    Field Size Type Description
    Type 1 uint8 0x0C
    Object ID 4 uint32 BE Entity whose weapons are being updated
    Weapon Strings 13 × variable length-prefixed Weapon data for each slot (as parseable strings)
    Ammo Array 1 52 13 × uint32 BE Primary ammo counts
    Ammo Array 2 52 13 × uint32 BE Secondary ammo counts
    Ammo Array 3 52 13 × uint32 BE Additional ammo data
    Ammo Array 4 52 13 × uint32 BE Additional ammo data

    ACK_WPNS (0x0D) Structure¶

    Acknowledgment packet confirming weapon state received:

    ACK_WPNS Packet Structure:
┌──────────┬──────────────┬────────────────────┬────────────────────┐
│  TYPE    │  OBJECT_ID   │  ACK_ARRAY_1       │  ACK_ARRAY_2       │
│  0x0D    │  (4 bytes)   │  (13 × 4 bytes)    │  (13 × 4 bytes)    │
└──────────┴──────────────┴────────────────────┴────────────────────┘

    Notes¶

    • 13 weapon slots matches the WEAPON_SLOTS_PER_VEHICLE = 13 constant from BEHAVIOR packet
    • Weapon strings likely contain floating-point reload timers or state values
    • ACK_WPNS confirms client received weapon state, allowing server to track sync status
