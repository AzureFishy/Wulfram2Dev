    Part 9.6: D_HANDSHAKE Deep Dive¶

    The UDP Connection Authentication Protocol¶

    Overview¶

    The D_HANDSHAKE packet is a critical component of Wulfram 2's UDP connection management system. Unlike the simple UDP packets used for game data (PRIORITY, VOICE, WEAPONS), D_HANDSHAKE operates within a specialized sub-protocol that handles UDP channel authentication, stream management, and connection state synchronization.

    This document provides a comprehensive technical analysis based on direct examination of the decompiled client executable (wulfram2_exe.c), specifically the sender function FUN_005052b0 (lines 181061-181131) and receiver function FUN_005053f0 (lines 181135-181253).

    The D_ Protocol Layer¶

    Wulfram 2 implements a layered UDP architecture:

    ┌─────────────────────────────────────────────────────────┐
│                    Application Layer                     │
│  (Game logic, player input, state management)           │
├─────────────────────────────────────────────────────────┤
│                   Game Data Layer                        │
│  PRIORITY(0x0B), VOICE(0x0A), WEAPONS(0x0C), etc.       │
├─────────────────────────────────────────────────────────┤
│              D_ Protocol Layer (Type 0x08)              │
│  D_HANDSHAKE(3), D_ACK(2), D_IGNORE(1), D_SET_START(4) │
├─────────────────────────────────────────────────────────┤
│                     UDP Transport                        │
│  Raw datagrams, connectionless                          │
└─────────────────────────────────────────────────────────┘

    The D_ protocol packets are encapsulated within UDP type 0x08 frames. The first byte of the D_ protocol payload specifies the D_ packet type:

    D_ Type Name Purpose
    0 (Reserved) Unknown/unused
    1 D_IGNORE Instructs receiver to ignore a stream
    2 D_ACK Acknowledges D_ protocol packets
    3 D_HANDSHAKE Stream configuration and authentication
    4 D_SET_START Sets stream start positions

    D_HANDSHAKE Purpose¶

    D_HANDSHAKE serves multiple critical functions:

    1. UDP Channel Authentication: Verifies that UDP traffic originates from the same client that established the TCP connection
    2. Stream Configuration: Transfers information about available communication streams (voice channels, data streams)
    3. Player Session Binding: Associates player indices with their stream allocations
    4. Sequence Synchronization: Establishes initial sequence numbers for reliable D_ protocol messaging

    Security Note: Without proper D_HANDSHAKE validation, the server would be vulnerable to UDP spoofing attacks where malicious actors could inject fake game data or voice packets.

    Field Descriptions¶

    Fixed Header Fields¶

    Field Offset Size Type Description
    D_Type 0 1 uint8 Always 0x03 for D_HANDSHAKE
    Sequence 1 4 uint32 Monotonically increasing sequence number. Client rejects packets with sequence ≤ last received
    Session_ID 5 4 uint32 Session identifier linking UDP to TCP connection
    Config_Value 9 4 uint32 Configuration value from server session state

    Stream Definition Fields¶

    Streams represent communication channels (voice chat rooms, data channels, etc.). Each stream definition contains:

    Field Size Type Description
    Stream_Name variable len-prefixed string Human-readable stream identifier (e.g., "Beacon Stream", "Voice Channel 1")
    Stream_Value 4 uint32 Stream-specific configuration value
    Stream_Indices 4 each uint32[] Array of indices this stream occupies

    String Encoding: Stream names use length-prefixed encoding: [uint16 length][bytes][null terminator]

    Player Entry Fields¶

    Field Size Type Description
    Active_Count 4 uint32 Number of active player entries to follow
    Player_Index 4 uint32 Index/slot of the player
    Player_Value 4 uint32 Player-specific value (possibly session or permission flags)

    D_ACK Response Structure¶

    When the client successfully processes D_HANDSHAKE, it responds with D_ACK:

    D_ACK Response (Client → Server):
┌──────────┬───────────┬──────────────┐
│ D_Type   │  Sub_Type │  Sequence    │
│  0x02    │   0x00    │  (4 bytes)   │
│ (1 byte) │  (1 byte) │   uint32     │
└──────────┴───────────┴──────────────┘
    Field Size Value Description
    D_Type 1 0x02 D_ACK packet type
    Sub_Type 1 0x00 Sub-type 0 = initial handshake ACK
    Sequence 4 uint32 Echoed timestamp for RTT calculation

    D_ACK Sub-Types¶

    The D_ACK packet has multiple sub-types for different purposes:

    Sub-Type Purpose Additional Fields
    0 Initial handshake ACK Sequence only
    1 D_IGNORE acknowledgment Stream index (u8), Value (u16)
    2 Extended ACK Sequence, Stream index (u8), Value (u16)

    Validation Requirements¶

    The client performs several validation checks on D_HANDSHAKE:

    1. Sequence Validation: received_sequence >= last_sequence
    2. Prevents replay attacks

    3. Packets with old sequences are silently discarded

    4. Stream Count Sanity Check: stream_count < max_streams

    5. Error message: "D_HANDSHAKE: insane stream_count"

    6. Prevents buffer overflow attacks

    7. Handshake State Check: Only process if not already handshaked

    8. Duplicate D_HANDSHAKE packets trigger re-ACK without reprocessing

    Implementation Guide¶

    Python Server Implementation¶

    import struct
import socket

class DHandshakePacket:
    """
    D_HANDSHAKE packet builder for Wulfram 2 server.

    This packet is sent within a UDP type 0x08 frame as part of
    the D_ protocol layer for connection authentication.
    """

    UDP_FRAME_TYPE = 0x08
    D_TYPE_HANDSHAKE = 0x03
    D_TYPE_ACK = 0x02

    def __init__(self, sequence: int, session_id: int, config_value: int):
        self.sequence = sequence
        self.session_id = session_id
        self.config_value = config_value
        self.streams = []  # List of (name, value, [indices])
        self.players = []  # List of (index, value)

    def add_stream(self, name: str, value: int, indices: list):
        """Add a stream definition to the handshake."""
        self.streams.append((name, value, indices))

    def add_player(self, player_index: int, player_value: int):
        """Add an active player entry."""
        self.players.append((player_index, player_value))

    @staticmethod
    def encode_string(text: str) -> bytes:
        """
        Encode a string in Wulfram 2 format.
        Format: [uint16 length][string bytes][null terminator]
        """
        string_bytes = text.encode('utf-8') + b'\x00'
        return struct.pack('>H', len(string_bytes)) + string_bytes

    def build(self) -> bytes:
        """
        Build the complete D_HANDSHAKE packet.

        Returns the D_ protocol payload (without UDP type byte).
        Caller must wrap in UDP type 0x08 frame.
        """
        packet = bytearray()

        # D_ packet type
        packet.append(self.D_TYPE_HANDSHAKE)

        # Fixed header: sequence, session_id, config_value
        packet.extend(struct.pack('>I', self.sequence))
        packet.extend(struct.pack('>I', self.session_id))
        packet.extend(struct.pack('>I', self.config_value))

        # Stream definitions
        for name, value, indices in self.streams:
            packet.extend(self.encode_string(name))
            packet.extend(struct.pack('>I', value))
            for idx in indices:
                packet.extend(struct.pack('>I', idx))

        # Player entries
        packet.extend(struct.pack('>I', len(self.players)))
        for player_idx, player_val in self.players:
            packet.extend(struct.pack('>I', player_idx))
            packet.extend(struct.pack('>I', player_val))

        return bytes(packet)

    def build_udp_frame(self) -> bytes:
        """Build complete UDP frame with type byte."""
        return bytes([self.UDP_FRAME_TYPE]) + self.build()


class DAckPacket:
    """D_ACK packet parser and builder."""

    D_TYPE_ACK = 0x02

    @staticmethod
    def build_handshake_ack(sequence: int) -> bytes:
        """Build D_ACK sub-type 0 (handshake acknowledgment)."""
        packet = bytearray()
        packet.append(0x08)  # UDP frame type
        packet.append(DAckPacket.D_TYPE_ACK)  # D_ACK
        packet.append(0x00)  # Sub-type 0
        packet.extend(struct.pack('>I', sequence))
        return bytes(packet)

    @staticmethod
    def parse(data: bytes) -> dict:
        """
        Parse incoming D_ACK packet.

        Returns dict with sub_type and relevant fields.
        """
        if len(data) < 3:
            raise ValueError("D_ACK packet too short")

        d_type = data[0]
        sub_type = data[1]

        result = {
            'd_type': d_type,
            'sub_type': sub_type
        }

        if sub_type == 0:
            # Handshake ACK: sequence only
            if len(data) >= 6:
                result['sequence'] = struct.unpack('>I', data[2:6])[0]
        elif sub_type == 1:
            # D_IGNORE ACK
            if len(data) >= 5:
                result['stream_index'] = data[2]
                result['value'] = struct.unpack('>H', data[3:5])[0]
        elif sub_type == 2:
            # Extended ACK
            if len(data) >= 9:
                result['sequence'] = struct.unpack('>I', data[2:6])[0]
                result['stream_index'] = data[6]
                result['value'] = struct.unpack('>H', data[7:9])[0]

        return result


# Example usage
def send_d_handshake(udp_socket, client_addr, session):
    """
    Send D_HANDSHAKE to establish UDP channel authentication.
    """
    handshake = DHandshakePacket(
        sequence=session.next_sequence(),
        session_id=session.session_id,
        config_value=session.config_flags
    )

    # Add voice channel streams
    handshake.add_stream("Team Voice", 1, [0, 1, 2, 3])
    handshake.add_stream("Proximity Voice", 2, [4, 5, 6, 7])

    # Add connected players
    for player in session.players:
        if player.is_active:
            handshake.add_player(player.index, player.permissions)

    packet = handshake.build_udp_frame()
    udp_socket.sendto(packet, client_addr)
    print(f"TX D_HANDSHAKE: {packet.hex(' ')}")


def handle_d_ack(data: bytes, client_addr):
    """Handle incoming D_ACK from client."""
    # Skip UDP frame type byte
    d_payload = data[1:]

    ack = DAckPacket.parse(d_payload)

    if ack['sub_type'] == 0:
        print(f"D_HANDSHAKE acknowledged, seq={ack.get('sequence')}")
        # Mark UDP channel as authenticated
        return True

    return False

    Error Conditions¶

    Error Cause Client Behavior
    Sequence too old seq < last_seq Silently discard packet
    Insane stream count stream_count >= max_streams Log error, discard packet
    Already handshaked Duplicate D_HANDSHAKE Re-send D_ACK only

    Security Considerations¶

    1. Sequence Numbers: The client tracks the last received sequence and rejects older packets. Server must maintain per-session sequence counters.

    2. Session Binding: The session_id field should match the TCP session to prevent cross-session attacks.

    3. Stream Count Limits: The client validates stream counts against a maximum. Server should not exceed this limit.

    4. UDP Source Validation: Even after D_HANDSHAKE, verify UDP source address matches the authenticated client.

    Debugging Tips¶

    1. Wireshark Filter: udp && data[0] == 0x08 && data[1] == 0x03 for D_HANDSHAKE packets

    2. Check Sequence: If D_HANDSHAKE is ignored, verify sequence is higher than previously sent

    3. Verify Framing: Ensure the packet is wrapped in UDP type 0x08, not sent raw

    4. Stream Count: Client has a maximum stream limit - check D_HANDSHAKE: insane stream_count in logs
