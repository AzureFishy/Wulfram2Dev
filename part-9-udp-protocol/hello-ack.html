    UDP D_HANDSHAKE / HELLO_ACK (0x08)¶

    Overview¶

    The UDP 0x08 packet serves two different purposes depending on direction:

    Direction Name Purpose
    Client → Server HELLO_ACK Simple keepalive/acknowledgment
    Server → Client D_HANDSHAKE Authentication handshake (complex structure)

    Discovery Credit: baffler identified that when sent TO the client, this packet is interpreted as D_HANDSHAKE and requires a more complex structure for UDP authentication.

    Authentication Purpose¶

    The D_HANDSHAKE mechanism appears to verify that UDP packets are coming from the same source as the established TCP connection. This prevents: - UDP spoofing attacks - Session hijacking via UDP - Unauthorized UDP traffic injection

    The server should validate that the UDP source address/port corresponds to an authenticated TCP session before accepting UDP traffic from that endpoint.

    D_HANDSHAKE (Server → Client) Structure¶

    ⚠️ WARNING: The D_HANDSHAKE structure is complex and not fully documented. baffler notes it "needs a lot of shit" — sending a malformed D_HANDSHAKE will likely cause client issues.

    When the server sends 0x08 TO the client, the client interprets it as D_HANDSHAKE, not a simple acknowledgment. This structure likely includes:

    When It's Sent¶

    Client → Server (HELLO_ACK): - After receiving TCP HELLO sub-type 1 (UDP port info) - Periodically as keepalive (approximately every 1000ms when no other UDP traffic)

    Server → Client (D_HANDSHAKE): - In response to client's first HELLO_ACK (to authenticate the UDP channel) - Possibly periodically for re-authentication

    HELLO_ACK Packet Structure (Client → Server)¶

    HELLO_ACK Structure:
┌──────────┬──────────────────┐
│  TYPE    │   PLAYER_ID      │
│  0x08    │   (4 bytes BE)   │
└──────────┴──────────────────┘
    Field Offset Size Type Description
    Type 0 1 uint8 Always 0x08
    Player ID 1 4 uint32 BE The sender's player ID

    Client Implementation¶

    def send_udp_hello_ack(udp_socket, server_addr, player_id):
    """
    Send UDP HELLO_ACK to establish/maintain UDP channel
    """
    packet = bytes([0x08])  # Type
    packet += struct.pack('>I', player_id)  # Player ID, big-endian

    udp_socket.sendto(packet, server_addr)

    Server Implementation¶

    def handle_udp_hello_ack(data, client_addr):
    """
    Handle incoming UDP HELLO_ACK from client
    """
    if len(data) < 5:
        return  # Invalid packet

    packet_type = data[0]
    player_id = struct.unpack('>I', data[1:5])[0]

    # Associate this UDP address with the player's session
    register_udp_endpoint(player_id, client_addr)

    # Send acknowledgment back
    response = bytes([0x08]) + struct.pack('>I', 0)  # Server player ID = 0
    udp_socket.sendto(response, client_addr)
