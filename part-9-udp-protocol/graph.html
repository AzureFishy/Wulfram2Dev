    UDP GRAPH_RESPONSE (0x12) and GRAPH_DATA (0x13)¶

    Overview¶

    GRAPH packets appear to be used for telemetry, debugging, or performance monitoring data. They transmit named data series with timestamped values.

    GRAPH_RESPONSE (0x12) Structure¶

    Response to a graph data request:

    GRAPH_RESPONSE Packet Structure:
┌──────────┬──────────────┬──────────┐
│  TYPE    │  NAME        │  EXISTS  │
│  0x12    │  (string)    │ (1 byte) │
└──────────┴──────────────┴──────────┘
    Field Size Type Description
    Type 1 uint8 0x12
    Name variable length-prefixed string Graph/data series name
    Exists 1 bool Whether the requested data exists

    GRAPH_DATA (0x13) Structure¶

    Actual graph data transmission:

    GRAPH_DATA Packet Structure:
┌──────────┬──────────────┬──────────────┬─────────────┬─────────────────────┐
│  TYPE    │  NAME        │  TIMESTAMP   │  COUNT      │  DATA_POINTS        │
│  0x13    │  (string)    │  (4 bytes)   │  (4 bytes)  │  (count × 8 bytes)  │
└──────────┴──────────────┴──────────────┴─────────────┴─────────────────────┘
    Field Size Type Description
    Type 1 uint8 0x13
    Name variable length-prefixed string Graph/data series name
    Timestamp 4 uint32 BE Time reference
    Count 4 uint32 BE Number of data points
    Data Points count × 8 array of doubles Data values (as strings, parsed to doubles)
