    UDP VOICE (0x0A)¶

    Overview¶

    The VOICE packet transmits voice chat audio data from clients to the server for distribution to other players. This is why the TCP HELLO sub-type 1 packet is critical — it establishes the UDP port used for voice.

    Packet Structure¶

    VOICE Packet Structure:
┌──────────┬──────────┬──────────┬──────────────┬────────────────┬─────────────┐
│  TYPE    │  FLAG_1  │  FLAG_2  │  PLAYER_ID   │  AUDIO_LENGTH  │  AUDIO_DATA │
│  0x0A    │ (1 byte) │ (1 byte) │  (4 bytes)   │   (2 bytes)    │  (variable) │
└──────────┴──────────┴──────────┴──────────────┴────────────────┴─────────────┘
    Field Offset Size Type Description
    Type 0 1 uint8 Always 0x0A
    Flag 1 1 1 uint8 Unknown flag
    Flag 2 2 1 uint8 Unknown flag
    Player ID 3 4 uint32 BE Sender's player ID
    Audio Length 7 2 uint16 BE Length of audio data
    Audio Data 9 variable bytes Compressed audio samples

    Server Handling¶

    The server typically: 1. Receives VOICE packet from sending client 2. Determines which players should hear this audio (proximity, team, etc.) 3. Forwards the VOICE packet to appropriate clients

    def handle_voice_packet(data, sender_addr):
    """Handle incoming voice packet"""
    if len(data) < 9:
        return

    flag1 = data[1]
    flag2 = data[2]
    player_id = struct.unpack('>I', data[3:7])[0]
    audio_len = struct.unpack('>H', data[7:9])[0]
    audio_data = data[9:9+audio_len]

    # Find nearby players and forward
    for recipient in get_voice_recipients(player_id):
        udp_socket.sendto(data, recipient.udp_addr)
