    Wulfram 2 Server Implementation Guide¶

    Part 3: Login Authentication¶

    Overview¶

    After the HELLO handshake, the client sends a LOGIN packet with the player's credentials. Your server must respond with LOGIN_STATUS to either accept or reject the login.

    CLIENT                                    SERVER
   │                                        │
   │  (HELLO handshake completed)           │
   │                                        │
   │───── LOGIN (username/password) ───────►│
   │                                        │
   │◄──── LOGIN_STATUS (success/fail) ──────│
   │                                        │
   │  If successful, client expects         │
   │  BEHAVIOR and TEAM_INFO next...        │
   │                                        │

    Receiving the LOGIN Packet (0x21)¶

    When the client sends LOGIN, you'll receive a packet containing credentials.

    What the Client Sends¶

    From the decompiled code (FUN_0046cea0, lines 77470-77472):

    thunk_FUN_00509440(in_EAX);   // Write 8-bit sub-type
FUN_00509600(param_2);         // Write username - LENGTH-PREFIXED string!
FUN_00509600(param_3);         // Write password - LENGTH-PREFIXED string!

    IMPORTANT: Username and password are length-prefixed strings, NOT simple null-terminated strings!

    LOGIN Packet Structure¶

    LOGIN (0x21) Structure:
┌──────────┬──────────────┬───────────┬──────────────────┬──────────────────┬────────────┐
│  TYPE    │   LENGTH     │ SUB_TYPE  │    USERNAME      │    PASSWORD      │  (extra)   │
│  0x21    │  (2 bytes)   │ (1 byte)  │ (len-prefixed)   │ (len-prefixed)   │  varies    │
└──────────┴──────────────┴───────────┴──────────────────┴──────────────────┴────────────┘

    Each string is encoded as: 

    ┌─────────────────┬──────────────────────────────┐
│  STRING_LENGTH  │  STRING DATA (with \0)       │
│  (2 bytes BE)   │  (LENGTH bytes)              │
└─────────────────┴──────────────────────────────┘

    Sub-Types¶

    Sub-Type Meaning Additional Data
    0x00 Normal login Just username/password
    0x02, 0x03, 0x04 Game service login Extra: count (32-bit) + N strings + final string

    Example: Player "ROOT" logging in with password "secret123"¶

    Building the packet:
├─ Type:            21                              (1 byte)
├─ Length:          00 14                           (20 bytes payload)
├─ Sub-type:        00                              (1 byte - normal login)
├─ Username length: 00 05                           (5 bytes = "ROOT" + null)
├─ Username data:   52 4F 4F 54 00                  ("ROOT\0")
├─ Password length: 00 0A                           (10 bytes = "secret123" + null)
└─ Password data:   73 65 63 72 65 74 31 32 33 00   ("secret123\0")

Complete packet:
21 00 14 00 00 05 52 4F 4F 54 00 00 0A 73 65 63 72 65 74 31 32 33 00
── ───── ── ───── ─────────────── ───── ───────────────────────────────
│    │   │    │         │           │              │
│    │   │    │         │           │              └─ "secret123\0" (10 bytes)
│    │   │    │         │           └─ Password length: 10
│    │   │    │         └─ "ROOT\0" (5 bytes)
│    │   │    └─ Username length: 5
│    │   └─ Sub-type: 0 (normal login)
│    └─ Payload length: 20 bytes (0x0014)
└─ Type: 0x21 (LOGIN)

Total: 23 bytes on wire

    Python: Parsing LOGIN Packet¶

    import struct

def decode_string(data, offset):
    """
    Decode a length-prefixed string from Wulfram 2 protocol.
    Returns (string, new_offset)
    """
    length = struct.unpack('>H', data[offset:offset+2])[0]
    string_data = data[offset+2:offset+2+length]
    # Remove null terminator for Python string
    text = string_data.rstrip(b'\x00').decode('utf-8')
    return text, offset + 2 + length

def parse_login_packet(data):
    """
    Parse a LOGIN packet received from the client.

    Args:
        data: Raw packet bytes (including type and length)

    Returns:
        dict with 'sub_type', 'username', 'password', and optionally 'extra_data'
    """
    # Verify packet type
    if data[0] != 0x21:
        raise ValueError(f"Not a LOGIN packet: 0x{data[0]:02x}")

    # Read length (big-endian)
    payload_length = struct.unpack('>H', data[1:3])[0]

    # Get payload
    payload = data[3:3+payload_length]

    # Parse sub-type (1 byte)
    sub_type = payload[0]
    offset = 1

    # Parse username (length-prefixed string)
    username, offset = decode_string(payload, offset)

    # Parse password (length-prefixed string)
    password, offset = decode_string(payload, offset)

    result = {
        'sub_type': sub_type,
        'username': username,
        'password': password
    }

    # For sub-types 2, 3, 4: parse additional game service data
    if sub_type in (0x02, 0x03, 0x04) and offset < len(payload):
        extra_count = struct.unpack('>I', payload[offset:offset+4])[0]
        offset += 4

        extra_strings = []
        for _ in range(extra_count):
            s, offset = decode_string(payload, offset)
            extra_strings.append(s)

        # One more string at the end
        final_string, offset = decode_string(payload, offset)

        result['extra_strings'] = extra_strings
        result['final_string'] = final_string

    return result

# Example usage:
# data = bytes.fromhex('21001400000552 4F 4F 5400000A736563726574313233 00')
# result = parse_login_packet(data)
# print(result)  # {'sub_type': 0, 'username': 'ROOT', 'password': 'secret123'}

    Sending LOGIN_STATUS (0x22)¶

    After receiving LOGIN, you must respond with LOGIN_STATUS to tell the client whether login succeeded.

    What the Client Expects¶

    From the decompiled code (FUN_0046db40, lines 77927-77935):

    void FUN_0046db40(param_1, param_2, param_3) {
    FUN_00509db0("LOGIN_STATUS", param_3);
    FUN_005092e0(&local_5);    // Read 8-bit flag
    FUN_005092e0(&local_4);    // Read 8-bit status
    FUN_00509dc0(param_3, param_2);

    FUN_004617f0(local_4);     // Process status code

    if ((char)local_4 == '\b') {   // Status == 8
        DAT_0067916b = local_5 == '\x01';  // Store flag
        DAT_0067916c = DAT_0067916b;
    }
}

    LOGIN_STATUS Packet Structure¶

    LOGIN_STATUS (0x22) Structure:
┌──────────┬──────────────┬───────────┬───────────────┐
│  TYPE    │   LENGTH     │   FLAG    │   STATUS      │
│  0x22    │   00 02      │ (1 byte)  │   (1 byte)    │
└──────────┴──────────────┴───────────┴───────────────┘
   1 byte     2 bytes        1 byte       1 byte

Total: 5 bytes on wire

    Status Codes¶

    From FUN_004617f0 (lines 69909-69961):

    Status Hex Client Action Description
    1 0x01 Shows password prompt Existing user - "Enter your password for this handle"
    2 0x02 Re-sends LOGIN with sub-type 3 Game service continuation
    3 0x03 Shows new password prompt New account - "This is a new handle. Supply a password"
    4 0x04 Proceeds to game Direct game entry (similar to 8)
    5 0x05 Shows handle prompt Login cleanup - "Welcome to/back to Wulfram!"
    6 0x06 Shows handle prompt Login cleanup - "Welcome to/back to Wulfram!"
    7 0x07 Shows handle prompt Login cleanup - "Welcome to/back to Wulfram!"
    8 0x08 Proceeds to team select SUCCESS - Login accepted!
    Other - Shows error message "XXX not handling some login_status return code XXX"

    The magic status code is 8 (0x08). This tells the client that login was successful and it should proceed to the team selection screen.

    Flag Byte¶

    The flag byte (first byte of payload) is stored in DAT_0067916b when status=8. - Set to 0x01 for a normal successful login - The exact meaning depends on server implementation, but 0x01 is standard

    Example: Successful Login Response¶

    Building LOGIN_STATUS for successful login:

Hex: 22 00 02 01 08
     ── ───── ── ──
     │    │   │  │
     │    │   │  └─ Status: 8 (SUCCESS!)
     │    │   └─ Flag: 1 (normal)
     │    └─ Length: 2 bytes
     └─ Type: 0x22 (LOGIN_STATUS)

Total: 5 bytes on wire

    Python Implementation¶

    def create_login_status(status=0x08, flag=0x01):
    """
    Create a LOGIN_STATUS response packet.

    Args:
        status: Status code (default 0x08 for success)
        flag: Flag byte (default 0x01 for normal)

    Returns:
        Complete packet bytes

    Status codes:
        1 = Existing user, prompt for password
        2 = Continue login flow (game service)
        3 = New account, prompt for new password
        4 = Direct game entry
        5, 6, 7 = Cleanup/restart login
        8 = SUCCESS - proceed to team select
    """
    packet_type = bytes([0x22])  # LOGIN_STATUS
    payload = bytes([flag, status])
    length = struct.pack('>H', len(payload))

    return packet_type + length + payload

# Successful login:
# create_login_status(status=0x08, flag=0x01)
# Returns: b'\x22\x00\x02\x01\x08'

# Failed login (restart):
# create_login_status(status=0x05, flag=0x00)
# Returns: b'\x22\x00\x02\x00\x05'

    Complete Login Flow Example¶

    Here's the complete server-side implementation of the login exchange:

    import struct

def encode_string(text):
    """Encode a length-prefixed, null-terminated string"""
    string_bytes = text.encode('utf-8') + b'\x00'
    return struct.pack('>H', len(string_bytes)) + string_bytes

def decode_string(data, offset):
    """Decode a length-prefixed string, returns (string, new_offset)"""
    length = struct.unpack('>H', data[offset:offset+2])[0]
    string_data = data[offset+2:offset+2+length]
    text = string_data.rstrip(b'\x00').decode('utf-8')
    return text, offset + 2 + length

def make_packet(packet_type, payload):
    """Wrap payload in a complete packet frame"""
    return bytes([packet_type]) + struct.pack('>H', len(payload)) + payload

class Wulfram2Server:
    PACKET_LOGIN = 0x21
    PACKET_LOGIN_STATUS = 0x22

    def send_packet(self, conn, data):
        """Send raw packet data to client"""
        conn.send(data)
        print(f"TX: {data.hex(' ')}")

    def recv_packet(self, conn):
        """Receive a complete packet from client"""
        # Read packet header (type + length)
        header = conn.recv(3)
        if len(header) < 3:
            return None, None, None

        packet_type = header[0]
        packet_length = struct.unpack('>H', header[1:3])[0]

        # Read payload
        payload = conn.recv(packet_length)
        print(f"RX: {header.hex(' ')} {payload.hex(' ')}")

        return packet_type, packet_length, payload

    def handle_login(self, conn):
        """
        Handle the login exchange after HELLO handshake.

        1. Receive LOGIN packet from client
        2. Validate credentials (or accept all for testing)
        3. Send LOGIN_STATUS response
        """

        # ═══════════════════════════════════════════════════════════
        # Step 1: Receive LOGIN packet
        # ═══════════════════════════════════════════════════════════
        packet_type, packet_length, payload = self.recv_packet(conn)

        if packet_type is None:
            print("Connection closed before LOGIN")
            return False

        # Verify it's a LOGIN packet
        if packet_type != self.PACKET_LOGIN:
            print(f"Expected LOGIN (0x21), got 0x{packet_type:02x}")
            return False

        # Parse LOGIN packet
        sub_type = payload[0]
        offset = 1

        # Decode username (length-prefixed string)
        username, offset = decode_string(payload, offset)

        # Decode password (length-prefixed string)
        password, offset = decode_string(payload, offset)

        print(f"Login attempt: sub_type={sub_type}, username='{username}', password='{password}'")

        # ═══════════════════════════════════════════════════════════
        # Step 2: Validate credentials
        # ═══════════════════════════════════════════════════════════
        # For testing, accept all logins
        # In production, check against your user database
        login_valid = True  # Accept all for now

        # ═══════════════════════════════════════════════════════════
        # Step 3: Send LOGIN_STATUS response
        # ═══════════════════════════════════════════════════════════
        if login_valid:
            # SUCCESS! Status = 8
            payload = bytes([0x01, 0x08])  # Flag=1, Status=8
            login_status = make_packet(self.PACKET_LOGIN_STATUS, payload)

            self.send_packet(conn, login_status)
            print("Login successful! Sent LOGIN_STATUS with status=8")

            # Expected bytes sent: 22 00 02 01 08
            return True
        else:
            # FAILURE! Status = 5 (cleanup/restart)
            payload = bytes([0x00, 0x05])  # Flag=0, Status=5
            login_status = make_packet(self.PACKET_LOGIN_STATUS, payload)

            self.send_packet(conn, login_status)
            print("Login failed! Sent LOGIN_STATUS with status=5")
            return False

    What Happens After LOGIN_STATUS?¶

    After sending LOGIN_STATUS with status=8, the client is now waiting for:

    1. BEHAVIOR packet (0x24) - The big configuration packet
    2. TEAM_INFO packet (0x28) - Team data for the selection screen

    The client will NOT show the team selection screen until it receives both of these packets!

    Quick Reference: LOGIN Packet Sizes¶

    Component Size Example
    Type 1 byte 0x21
    Length 2 bytes 0x0014 (20)
    Sub-type 1 byte 0x00
    Username 2 + len + 1 bytes 00 05 52 4F 4F 54 00
    Password 2 + len + 1 bytes 00 0A 73 65 63 72 65 74 31 32 33 00

    Payload formula: 1 + (2 + username_len + 1) + (2 + password_len + 1)

    Example: username="ROOT" (4 chars), password="secret123" (9 chars) - Payload = 1 + (2 + 4 + 1) + (2 + 9 + 1) = 1 + 7 + 12 = 20 bytes

    Troubleshooting¶

    Problem: Client Stuck at Login Screen¶

    Cause: LOGIN_STATUS not sent, or wrong status code.

    Solution: Make sure you're sending 22 00 02 01 08 after receiving LOGIN.

    # This exact sequence is required:
conn.send(bytes.fromhex('22 00 02 01 08'))

    Problem: Client Shows Error After Login¶

    Cause: Status code other than 8, or packets sent in wrong order.

    Solution: 1. Use status code 8 for success 2. Make sure you completed HELLO handshake before LOGIN 3. Send BEHAVIOR and TEAM_INFO after LOGIN_STATUS

    Problem: Server Fails to Parse LOGIN¶

    Cause: Expecting null-terminated strings instead of length-prefixed strings.

    Solution: Use proper length-prefixed string decoding:

    # WRONG - Don't do this:
strings = payload[1:].split(b'\x00')  # ← WRONG!

# CORRECT - Do this:
username, offset = decode_string(payload, 1)
password, offset = decode_string(payload, offset)

    Problem: Nothing Happens After LOGIN_STATUS¶

    Cause: Client is waiting for BEHAVIOR packet.

    Solution: Immediately send BEHAVIOR (0x24) after LOGIN_STATUS. The client won't proceed without it.

    Common Mistakes¶

    Mistake Symptom Fix
    Null-terminated strings Parse failure Use length-prefixed string decoding
    Wrong payload length Packet rejected Recalculate: 1 + (2+len+1) per string
    Status code not 8 Client shows prompts Use status=8 for successful login
    Missing BEHAVIOR packet Client hangs Send BEHAVIOR immediately after LOGIN_STATUS
    Wrong byte order Parse failure Use big-endian (>H) for length prefix
